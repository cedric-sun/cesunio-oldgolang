Golang template and consideration for modularization

Note that essentially the templating system is designed for full featured dynamic sites, thus it's quite unnatural to separate the site compiler itself from whatever site you are going to build. Some idea I'm having now is to totally decouple the logic of collecting data (view) for a specific template output into a single .so file which will be written by client and whose name will be specified in site, and load it as a golang plugin and parse some pre-defined symbol (or defined by client's site file)
